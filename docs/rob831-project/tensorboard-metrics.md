# TensorBoard Metrics Reference

This document describes all metrics logged during PPO training on the RoboEval LiftPot task.

## Results Directory Structure

Each SLURM job writes results into `results/<job-name>_<job-id>/`, keeping experiments separate.

```
results/
└── <job-name>_<job-id>/              # e.g. test-roboeval-liftpot_12345
    ├── tensorboard/
    │   ├── config.yaml               # Training configuration snapshot
    │   └── events.out.tfevents.*     # TensorBoard event logs
    ├── video/
    │   ├── train/                    # Training rollout videos (if enabled)
    │   └── eval/
    │       └── seed_<N>/             # Eval rollout recordings per seed
    │           ├── 0.mp4
    │           └── ...
    └── mean_return_vs_steps.png      # Generated by plot script
```

SLURM log files (`.out`/`.err`) are kept separately in `slurm/` via `LOG_DIR`.

## Environment Metrics (`env/` and `eval/`)

Logged per epoch for training envs (`env/`) and every `val_check_interval=10` epochs for eval envs (`eval/`).

| Tag | Description |
|-----|-------------|
| `return` | Mean episodic return across all parallel envs |
| `reward` | Mean per-step reward |
| `episode_len` | Mean episode length (max 200 steps) |
| `success_once` | Fraction of episodes where the task was completed at least once |
| `num_trajectories` | Number of trajectories collected (32 train, 16 eval) |

## Training Metrics — Actor (`train/actor/`)

Logged every epoch after PPO updates (`update_epoch=8` gradient steps per epoch).

| Tag | Description |
|-----|-------------|
| `policy_loss` | PPO clipped surrogate objective loss |
| `total_loss` | Combined loss (policy + entropy) |
| `entropy_loss` | Policy entropy — measures exploration; decreasing means the policy is becoming more deterministic |
| `approx_kl` | Approximate KL divergence between old and new policy — monitors how much the policy changes per update |
| `clip_fraction` | Fraction of samples where the importance sampling ratio was clipped (by `clip_ratio_high=0.2`) |
| `ratio` / `ratio_abs` | Importance sampling ratio (pi_new / pi_old) — should stay near 1.0 |
| `clipped_ratio` | Mean clipped ratio value |
| `dual_cliped_ratio` | Fraction affected by dual clipping (lower bound, `clip_ratio_c=3.0`) |
| `grad_norm` | Gradient L2 norm after clipping (`clip_grad=0.5`) — spikes indicate training instability |
| `lr` | Actor learning rate (constant at `3e-4`) |

## Training Metrics — Critic (`train/critic/`)

| Tag | Description |
|-----|-------------|
| `value_loss` | MSE loss for the value function head |
| `explained_variance` | How well the value function predicts actual returns (1.0 = perfect, 0.0 = no better than mean, negative = worse than mean) |
| `value_clip_ratio` | Fraction of value function updates that were clipped (`value_clip=1.0`) |
| `lr` | Critic learning rate (constant at `3e-4`) |

## Rollout Buffer Statistics (`rollout/`)

Statistics computed over the rollout buffer before each training epoch.

| Tag | Description |
|-----|-------------|
| `rewards` | Mean reward across the buffer |
| `returns_mean/max/min` | GAE return statistics (`gamma=0.99`, `lambda=0.95`) |
| `advantages_mean/max/min` | Advantage estimates (normalized when `normalize_advantages=true`) |

## Timing Metrics (`time/`)

Wall-clock time (seconds) for each phase of the training loop.

| Tag | Description |
|-----|-------------|
| `step` | Total time per training epoch |
| `sync_weights` | Time to sync model weights from actor to rollout worker |
| `generate_rollouts` | Time to collect rollout data from envs |
| `cal_adv_and_returns` | Time to compute advantages and returns |
| `env/env_interact_step` | Time per env interaction step |
| `env/interact` | Total env interaction time |
| `rollout/predict` | Time for policy forward pass during rollout |
| `rollout/generate_one_epoch` | Total time for one rollout epoch |
| `actor/run_training` | Time for PPO gradient updates |
| `eval` | Time for evaluation rollouts (logged every 10 epochs) |

## Plotting

Run the plotting script from the RLinf root, passing the results directory for the experiment:

```bash
python3 scripts/plot_return.py ../results/<job-name>_<job-id>
```

This produces `mean_return_vs_steps.png` inside that results directory, showing mean return vs environment steps for both train and eval.

### Step Conversion

TensorBoard logs steps as **epochs** (0–80). To convert to environment steps:

```
env_steps = (epoch + 1) × total_num_envs × max_steps_per_rollout_epoch
          = (epoch + 1) × 32 × 200
          = (epoch + 1) × 6400
```

## Training Configuration Summary

- **Task**: LiftPot (lift pot by gripping both handles)
- **Algorithm**: PPO with GAE, actor-critic loss
- **Policy**: MLP (hidden_dim=256, obs_dim=38, action_dim=16)
- **Train envs**: 32 parallel, **Eval envs**: 16 parallel
- **Max epochs**: 200, **Max episode steps**: 200
- **Eval interval**: every 10 epochs
- **Batch size**: 64 global, 32 micro
- **PPO updates per epoch**: 8
